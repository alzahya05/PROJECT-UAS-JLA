### Prinsip Struktur:
- **Route group per modul** → folder terpisah (`edukasi/`, `aksesibilitas/`, `komunitas/`, `pengaturan/`)
- **`layout.tsx` per route group** → shared UI (navbar modul) tanpa re-mount
- **`components/` di root `app/`** → reusable, bisa di-import dari mana saja
- **`hooks/`** → custom hooks terisolasi
- **`lib/`** → utilities tanpa UI

## 2. Server vs Client Components

### Server Components (default, tanpa directive):
| Komponen | Alasan |
|---|---|
| `app/layout.tsx` | Root layout, metadata static |
| `app/page.tsx` | Homepage navigasi static |
| `app/*/layout.tsx` | Shared layout per modul |
| `components/layout/Navbar.tsx` | Navigasi static, SEO-friendly |
| `components/layout/Sidebar.tsx` | Menu sidebar static |
| Semua `page.tsx` yang content-heavy | Artikel, kelas, profil — render di server |

### Client Components (`"use client"`):
| Komponen | Alasan |
|---|---|
| `components/layout/FAB.tsx` | Floating Action Button — butuh onClick, state open/close |
| `components/shared/SpeechRecognizer.tsx` | Web Speech API — `window.SpeechRecognition` |
| `components/shared/AudioVisualizer.tsx` | Web Audio API — `AudioContext`, microphone |
| `components/shared/CameraFeed.tsx` | `navigator.mediaDevices.getUserMedia()` |
| `components/shared/VideoPlayer.tsx` | HTML5 `<video>` controls, play/pause state |
| `components/shared/QuizCard.tsx` | Interactive state: selected answer, score, timer |
| `components/shared/ProgressTracker.tsx` | `localStorage` access, state update |
| `/aksesibilitas/speech-to-text/page.tsx` | Full page pakai Web Speech API |
| `/aksesibilitas/sound-detector/page.tsx` | Full page pakai Web Audio API |
| `/aksesibilitas/scanner/page.tsx` | Full page pakai kamera |
| `/edukasi/latihan/page.tsx` | Kuis interaktif, progress state |
| `/pengaturan/notifikasi/page.tsx` | Toggle/checkbox state |

### Aturan umum:
- **Default ke Server Component** → lebih cepat, SEO-friendly
- **Pindah ke Client hanya saat butuh**: `useState`, `useEffect`, event handler, Web API (`window`, `navigator`, `document`)
- **Komponen child yang client → parent tetap server** ✅
- **Pisahkan logic client ke file terpisah** → jangan tulis `"use client"` di page-level kalau bisa dipecah ke child component

## 3. Rekomendasi UI Library & Aksesibilitas

### Stack Utama:

| Library | Fungsi | Aksesibilitas |
|---|---|---|
| **Tailwind CSS v4** | Utility-first CSS | ✅ Sudah terinstall |
| **shadcn/ui** | Pre-built accessible components | ✅ Built on Radix Primitives, WCAG compliant |
| **Radix UI Primitives** | Headless accessible components | ✅ WCAG 2.1, keyboard navigation, ARIA labels |
| **Lucide React** | Icon library | ✅ Accessible SVG icons |
| **clsx / tailwind-merge** | Class name utilities | ✅ Dibutuhkan shadcn |

### Libraries Khusus Aksesibilitas:

| Library | Fungsi |
|---|---|
| **@radix-ui/react-dialog** | Modal yang screen-reader friendly |
| **@radix-ui/react-dropdown-menu** | Dropdown menu accessible |
| **@radix-ui/react-toast** | Notifikasi visual (penting untuk user tunarungu!) |
| **@radix-ui/react-tabs** | Tab navigation accessible |
| **@radix-ui/react-switch** | Toggle switch untuk settings |
| **@radix-ui/react-checkbox** | Checkbox accessible |
| **@radix-ui/react-slider** | Slider untuk volume/frekuensi |

### Web API Libraries (untuk fitur aksesibilitas):

| Library | Fungsi |
|---|---|
| **MediaDevices API** (native) | Kamera & mikrofon access |
| **Web Speech API** (native) | Speech recognition & synthesis |
| **Web Audio API** (native) | Audio frequency analysis |

> **Catatan:** Web API di atas native browser — tidak perlu library tambahan, cukup type definition.

## 4. Perintah pnpm Install

```bash
# UI Framework & Components
pnpm add @radix-ui/react-dialog @radix-ui/react-dropdown-menu @radix-ui/react-toast @radix-ui/react-tabs @radix-ui/react-switch @radix-ui/react-checkbox @radix-ui/react-slider @radix-ui/react-slot

# Icons & Utilities
pnpm add lucide-react clsx tailwind-merge

# Animations (optional, untuk micro-interactions)
pnpm add framer-motion

# Dev Dependencies
pnpm add -D @types/node
```

### Mengapa library ini:
- **Radix Primitives** → headless, fully accessible, kompatibel Next.js
- **Lucide React** → ringan, accessible SVG icons
- **clsx + tailwind-merge** → standard utilities untuk conditional classes (dibutuhkan shadcn)
- **Framer Motion** → animasi halus untuk UX yang lebih baik (visual feedback penting untuk user tunarungu)

## 5. Implementation Steps

### Step 1: Install dependencies
```bash
pnpm add @radix-ui/react-dialog @radix-ui/react-dropdown-menu @radix-ui/react-toast @radix-ui/react-tabs @radix-ui/react-switch @radix-ui/react-checkbox @radix-ui/react-slider @radix-ui/react-slot lucide-react clsx tailwind-merge framer-motion
```

### Step 2: Setup shadcn/ui (manual atau CLI)
- Buat `components/ui/` folder
- Copy & customize shadcn components: Button, Card, Dialog, Toast, Switch, Checkbox

### Step 3: Buat shared utilities
- `lib/utils.ts` → fungsi `cn()` untuk merge className
- `lib/constants.ts` → menu items, routes, app metadata

### Step 4: Buat root layout components
- `components/layout/Navbar.tsx` — navigasi utama
- `components/layout/FAB.tsx` — Floating Action Button

- `app/edukasi/layout.tsx`
- `app/aksesibilitas/layout.tsx`
- `app/komunitas/layout.tsx`
- `app/pengaturan/layout.tsx`
